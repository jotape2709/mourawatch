Scan de Servidores

Teste de alcance a servidores públicos via requisição HTTP do navegador

Como funciona o teste?
O navegador tenta alcançar cada servidor enviando uma requisição HTTP real. Se o servidor responder (mesmo com erro 4xx/5xx), ele está acessível. Se a requisição falhar completamente (CORS bloqueado, timeout, rede recusada), ele é marcado como inacessível pelo navegador.

Limitação importante: Browsers bloqueiam requisições cross-origin sem cabeçalho CORS. Isso não significa que o servidor está offline — significa que ele não foi projetado para aceitar chamadas diretas de páginas web. Use ferramentas como ping, curl ou Postman para testes sem essa restrição.
Por que não consigo "pingar" servidores direto do navegador?
1
Política de Same-Origin (CORS)
Navegadores aplicam uma política de segurança chamada CORS. Uma página web só pode buscar dados de outro domínio se esse domínio explicitamente permitir, via cabeçalho Access-Control-Allow-Origin. Sem isso, a requisição é bloqueada pelo próprio browser — não pelo servidor.
2
ICMP (ping) não existe no JavaScript
O protocolo PING usa ICMP, que opera abaixo do HTTP. Navegadores só têm acesso à camada HTTP/HTTPS. Logo, é tecnicamente impossível fazer um "ping real" de dentro de uma página web — só é possível fazer requisições HTTP.
3
O que o MouraWatch faz então?
Realiza requisições HTTP reais aos endpoints, medindo tempo de resposta, código de status e disponibilidade. APIs com CORS aberto respondem corretamente. Servidores sem CORS retornam erro de rede — mas registramos a tentativa e o resultado.
4
Para monitoramento de infra real
Você precisaria de um agente backend (ex: Prometheus Node Exporter, Zabbix, Uptime Kuma) rodando no servidor, expondo métricas via uma API com CORS habilitado — que então este dashboard consultaria.
